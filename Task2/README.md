# Task2

## Основные домены  

- Домен клиентский
Управление взаимодействием с клиентами, включая хранение данных о них, историю взаимодействий и обслуживание.
- Домен сотрудника
Управление взаимодействием с клиентами, данные и действия сотрудника, историю действий по обслуживанию запросов клиента.
- Домен медицинский
Медицинские карты, истории болезней, данные исследований.
- Домен финансовый
Финансовая история клиентов, счета, данные по кредитам.
- Домен аналитический
Витрина данных для самообслуживания, централизованная аналитика.

## Обоснование

### Модульность  
Разделение системы на модули (при использовании DDD) делает итоговую систему более гибкой и модульной, что позволит раздельно вести разработки и обновлять модули независимо друг от друга.

### Масштабируемость  
Разделение системы на модули аозволяет каждому направлению развиваться независимо, и масштабироваться также независимо. 

### Безопасность  
Разделение данных по доменам позволяет через введение контроля доступа согласно ролевым моделям, созданным для соответствующих модулей и направлений повысит безопасность системы.
